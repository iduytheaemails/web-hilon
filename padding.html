Pengantar program

Program inventory divisi padding adalah program yang digunakan untuk membuat label produk barang yang dibuat oleh bagian produksi padding, selain itu program ini akan mencatat data produksi dan hasil produksinya. Dari setiap barang produksi yang telah diberi label produk akan secara otomatis menambahkan jumlah stok barang di bagian gudang, jadi setiap saat jumlah stok barang akan berubah secara reeltime sesuai dengan mutasi yang sedang berjalan.

Fitur

Program inventory divisi padding ini dirancang agar dapat digunakan semudah mungkin oleh operator dalam pengoresinnya. Dan program ini memiliki fitur-fitur sebagai berikut :

1. Berbasis OS Windows (7, 8, 8.1, 10)

2. Server database menggunakan mysql

3. Untuk modul dan query menggunakan MS ACCESS

4. Multi user, sederhana dan mudah digunakan

5. Data aman karena berada di sisi server

SKEMA PROGRAM PADDING

PROSEDUR PROGRAM PADDING

Prosedur registrasi data produksi

Dalam proses pengiriman barang diperlukan adanya data tujuan kemana atau kepada siapa barang itu akan dikirimkan, diperlukan juga nomor kendaraan yang akan membawakan barang tersebut. Selanjutnya program akan membuatkan nomor referensi pengiriman atau disebut nomor packing list berserta tanggal dan jam pengiriman.

Prosedur input data berat aktual

Setelah data-data tujuan pengiriman dibuat selanjutnya tinggal memasukan data nomor barang yang akan dikirim ke dalam daftar barang yang akan dikirim, dengan cara memindai/scan kode batang (barcode) atau menuliskan langsung kode nomor produk dari barang tersebut. Adapun cara yang lainnya untuk memasukan kode barang yaitu meng-dobel klik di field kode SAP yang berada dibagian tabel stok barang gudang. Setiap barang yang dimasukan ke daftar barang yang akan dikirim secara otomatis akan mengurangi jumlah stok barang di gudang. Untuk pembatalan, jika barang yang telah masuk daftar barang kirim tenyata akan dibatalkan maka tingal menghapus kode barang yang dimaksud itu dari daftar kirim barang dan program akan menambahkan kembali jumlah stok barang digudangnya.

Pencetakan label produk

Jika barang yang akan dikirim sudah masuk ke dalam daftar barang yang dikirim, selanjutnya tinggal mencetak packing list sebagai bukti pengeluaran barang dari gudang, tetapi terlebih dahulu untuk mengatur opsi-opsi pencetakan untuk menentukan jenis satuan barang yang akan digunakan, ada dua pilihan untuk penggunaan satuan perhitungan jumah barang yang dikirim, yaitu berdasarkan satuan setandar UOM atau berdasarkan satuan yang kita atur sendiri, misalkan satuan barang UOM adalah KGM tetapi konsumen ingin menggunakan satuan barang YRD untuk menghitung jumlah barang itu, maka kita dapat mengatur opsi pencetakannya.

Buat laporan produksi

Jika barang yang akan dikirim sudah masuk ke dalam daftar barang yang dikirim, selanjutnya tinggal mencetak packing list sebagai bukti pengeluaran barang dari gudang, tetapi terlebih dahulu untuk mengatur opsi-opsi pencetakan untuk menentukan jenis satuan barang yang akan digunakan, ada dua pilihan untuk penggunaan satuan perhitungan jumah barang yang dikirim, yaitu berdasarkan satuan setandar UOM atau berdasarkan satuan yang kita atur sendiri, misalkan satuan barang UOM adalah KGM tetapi konsumen ingin menggunakan satuan barang YRD untuk menghitung jumlah barang itu, maka kita dapat mengatur opsi pencetakannya.

A. Registrasi Data Produksi

Jalankan program Inventory Padding dan program akan menampikan form utama seperti gambar di bawah ini.

AREA : A pada gambar

1. Isi bagian Shift dengan kelompok shift kerja

AREA : B pada gambar

2. Isi bagian customer dengan nama costumer
3. Isi kode item dengan kode SAP item yang diproduksi
4. Isi berat polybag yang digunakan

AREA : C pada gambar

5. Isi jml bale dengan jumlah roll yang akan dibuat
6. Isi No.Roll dengan urutan ke-berapa akan dimulai

Contoh: Akan diproduksi item A sebanyak 10 roll dan dimulai dari nomor roll 1, maka isikan jml bale dengan angka 10 dan isi no.roll dengan angka 1. Tetapi jika nomor urutan rollnya akan dimulai dari angka 5 maka isi no.roll dengan angka 5 dan program akan mencetak label sebanyak 10 lembar dengan urutan roll ke 5 sampai urutan roll ke 15.

B. Input Data Berat Aktual

Produk yang telah selesai proses produksi, selanjutnya diberi/dibungkus dengan polybag lalu ditimbang untuk mengetahui berat aktualnya, kemudian hasil timbangannya itu di-inputkan ke program sebagai data berat aktual produk. Selanjutnya tinggal mencetak label produksi dengan menekan tombol "OK"

C. Cetak Label Produk

Setelah tombol "OK" selanjutnya program akan menampikan form "PRINT LABEL" seperti di bawah ini

Selanjutnya tekan tombol "PRINT !" untuk memulai proses pencetakan label. Kemudian form ini akan tertutup otomatis dan kembali ke form sebelummnya untuk menginput kembali data berat aktual dari produk selanjutnya dan kemudian akan dicetak labelnya lagi. Proses ini akan terus diulang sampai produksi selesai.
